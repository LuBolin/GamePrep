## Source conflicts and cautions

- `SRC-EPIC-003` uses broad “smart pointers” wording. `SRC-EPIC-009` distinguishes ordinary shared pointers from UObject-specific pointer wrappers; the dedicated pointer reference controls pointer selection.
- `SRC-EPIC-005` is evidence for an experimental UE5.6 mode, not evidence that every UE5.6 project runs incremental reachability.
- Search and current documentation navigation may resolve a versioned URL to a newer maintained page. Claims that depend on exact API availability require confirmation against UE5.6 API/source before being labelled fully covered.

- Specifiers are not decorative. They are contracts consumed by different systems: UHT, editor details panels, Blueprint compiler, serialisation, config, SaveGame, duplication, networking, GC, cook and tooling. A flag is incomplete until the consuming system and lifecycle are named.

- The maintained Epic API site can be hard to pin to an exact minor version for Core templates. Treat `TOptional`, `TVariant`, `TFunctionRef`, `FGCObject` and async-action details as branch-check items before copying signatures into UE5.3-UE5.6 code.

- Pak, IoStore (`.utoc`/`.ucas`), chunking, DDC/Zen, World Partition and loading trace details evolve across UE5/platforms. Verify exact project settings and packaged output rather than inferring from a maintained page.
